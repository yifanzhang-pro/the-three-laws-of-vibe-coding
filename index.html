The Three Laws of Vibe Coding

Generate no slop. Deslop with intent. Preserve velocity.
In that order.

By Yifan Zhang

The Laws

Quality is the constraint. Intent is the compass. Velocity matters only after both are satisfied.

01

Do not generate slop.

Vibe coding must not generate slop or, through inaction, allow slop to enter and harm the codebase.

02

Deslop with intent.

Vibe coding must deslop generated code according to developer intent, project conventions, and testing requirements, except where such requirements would conflict with the First Law.

03

Preserve velocity.

Vibe coding must preserve development velocity, as long as doing so does not generate slop, obstruct deslopping, or conflict with the First or Second Law.

Fast is useful. Correct is essential. Maintainable is the standard.

The ordering is deliberate: every later law yields to the laws before it.
